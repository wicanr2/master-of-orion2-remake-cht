## 一、經濟總覽:錢與生產分離

MOO2 與前作 MOO1 最大的經濟差異:**金錢(BC, Bank Credits)與工業生產(PP, Production)是兩套獨立帳本**。工業產能主要拿來蓋建築/造艦,不會「自動變成錢」;帝國的錢有其獨立來源(人頭稅、賣糧、貿易品、特殊礦藏),再靠稅率把一部分工業「抽稅」轉成錢。玩家因此可以「工業全開造東西、稅率設 0 也仍有基礎收入」——這條設計是 MOO2 經濟手感的骨架。

四類「錢」的來源(手冊 p.37 國庫指標把收入來源列為稅、賣糧、貿易品三項,加上人頭基礎收入):

| 來源 | 性質 | 手冊頁 |
|---|---|---|
| 人頭基礎收入(per-capita income) | 每人口每回合自動產生,與工業無關 | p.20 / p.22 |
| 工業稅(Industrial Tax) | 把工業產能按稅率抽成、1:1 換 BC | p.37 / p.168 |
| 貿易品(Trade Goods) | 整座殖民地把建造改設貿易品,2:1 換 BC | p.70 |
| 賣餘糧 + 特殊礦藏 | 剩食每單位 0.5 BC;金/寶石星定額加成 | p.25 / p.61 |

支出面(手冊 p.37):建築維護費、艦艇/指揮評等超支、領袖薪資、運輸艦維護。

## 二、BC 收入模型(核心)

### 2.1 人頭基礎收入(per-capita income)

**每個人口單位每回合自動產生基礎 1 BC**,與該人口被指派為農夫/工人/科學家無關,也與工業產能完全分離。手冊三處佐證此量存在:

- p.20:種族 Money 特質說明「Imperial monetary penalties cannot reduce taxation possibilities below **zero per population unit**」——證實收入以「每人口單位」為基準單位。
- p.22:民主政府「+50% ... **per capita (tax) income**」——證實有「per capita income」這個量。
- p.16:Gnolam(諾蘭姆)「each unit of Gnolam population generates an **additional 1 BC** per turn」——「額外」一詞反證基礎值本身存在。

> remake 以半 BC 為內部單位:`gamedata.BaseIncomePerPopHalfBC = 2`(= 1.0 BC/人)。

### 2.2 種族 Money 特質(在基礎上加減)

自訂種族的「Money」picks 與內建種族天賦,在每人基礎 1 BC 上加減(手冊 p.20 定性;數值為社群/自訂種族點數表慣例):

| Money 特質 | 每人每回合 BC | 說明 |
|---|---|---|
| 差(Low, −1 pick) | 0.5 | 基礎 1 − 0.5 |
| 標準 | 1.0 | 基礎值 |
| 佳(High, +1) | 1.5 | 基礎 1 + 0.5 |
| 優(+2) | 2.0 | 基礎 1 + 1 |
| 諾蘭姆(Gnolam) | 2.0 | 手冊逐字「額外 +1 BC/人」(等同 money +2 pick),p.16 |

下限鎖 0/人(手冊 p.20:「cannot reduce below zero per population unit」)。remake:`ColonyState.IncomePerPop`(半 BC delta),諾蘭姆 = +2 半 BC。

### 2.3 工業稅(Industrial Tax)

手冊 p.37 / p.168:

| 參數 | 值 |
|---|---|
| 稅率範圍 | 0% ~ 50% |
| 級距 | 10%(合法值:0/10/20/30/40/50) |
| 換算率 | **1:1**(每 1 點被抽稅的工業 = 1 BC) |

**代價**:每提高一級稅率,全帝國每座殖民地的可用產能同步下降。p.168 例:稅率 50% → 一半工業被抽走當稅、只剩一半能拿去蓋東西。

```
稅收 BC          = 該殖民地工業產能 × 稅率%        (向下取整)
剩餘可建造工業    = 工業產能 − 稅收 BC
```

稅率**不影響士氣**(手冊 p.168 明確)。稅的 1:1 換算率優於貿易品的 2:1(p.168),但稅是「全帝國每座殖民地都減產」,貿易品是「把單一殖民地整座轉成商業」——取捨不同。

### 2.4 貿易品(Trade Goods)

把某殖民地的建造項改設為「貿易品」,該殖民地當回合的淨工業整包不蓋建築,改換成 BC(手冊 p.70):

| 種族 | 換算率 |
|---|---|
| 一般 | 2 工業 → 1 BC |
| Fantastic Trader(如諾蘭姆) | 1 工業 → 1 BC(等效 2 倍) |

> 手冊敘述小矛盾:p.25 說 Fantastic Trader「貿易品收入 +50%」,p.70 則直接寫「1:1」(= 2 倍)。remake 採 p.70 的直接換算比(1:1),`+50%` 敘述未另外疊加,標為待查證(見 `income.go` 註解)。

### 2.5 賣餘糧與特殊礦藏

| 項目 | 一般種族 | Fantastic Trader | 手冊頁 |
|---|---|---|---|
| 每單位剩餘糧食 | 0.5 BC | 1 BC | p.25 |
| 金礦星(Gold) | 該殖民地 +5 BC/回合(定額) | 同 | p.61 |
| 寶石星(Gems) | 該殖民地 +10 BC/回合(定額) | 同 | p.61 |

餘糧只在「淨盈餘 > 0」時計入(饑荒的負盈餘由人口停止成長懲罰處理,不倒扣 BC)。金/寶石屬「系統特殊」(§5),是探索時隨機出現的定額收入,屬「該殖民地所有來源」之一,會被下節的建築倍率一併放大。

### 2.6 建築與政府倍率

**建築倍率(逐殖民地,可疊加)** — 對該殖民地「所有來源 BC 收入」按百分比放大:

| 建築 | 加成 | 維護 | 手冊頁 |
|---|---|---|---|
| 太空港(Spaceport) | +50% | 1 BC | p.79 |
| 行星證券交易所(Planetary Stock Exchange) | +100% | 2 BC | p.93 |

兩者同時存在時相加(+150%)。remake:`ColonyState.IncomeBonusPercent`(領袖 Trader 技能也進同一欄位,可再疊加)。

**政府倍率(帝國層級,對加總後的 money 收入套一次)** — 來源 `MANUAL_150.html`(換算式 `value × 5 = 加成%`):

| 政府 | money 加成 |
|---|---|
| 民主(Democracy) | +50% |
| 聯邦(Federation) | +75% |

其餘政府手冊未列 money 加成(= 0)。

### 2.7 殖民地每回合 BC 組合公式

把上述各項按 remake `engine/empire.go` 的結算順序串起來,一座殖民地當回合對帝國國庫的貢獻:

```
── 逐殖民地層 ──────────────────────────────
perCapita = 人口 × max(0, 每人基礎收入)          # 基礎1BC ± 種族Money特質
tax       = 淨工業 × 稅率%                       # 1:1,向下取整
foodRev   = max(0, 餘糧) × 0.5(Trader ×1)
tradeRev  = 若設貿易品:淨工業 × 0.5(Trader ×1),否則 0
special   = 金礦 +5 / 寶石 +10(若該星有)

殖民地小計 = (perCapita + tax + foodRev + tradeRev + special)
           × (1 + 建築加成%/100)               # 太空港+50、證交所+100,可疊加

── 帝國層 ──────────────────────────────────
帝國 money = Σ(各殖民地小計) × (1 + 政府money加成%/100)   # 民主+50、聯邦+75
淨 BC      = 帝國 money − 建築維護費 − 指揮評等超支(§7)
                       − 運輸艦維護費(§7)
```

要點:①「淨工業」已是士氣/重力調整後的產出,故收入端不再重複套士氣(避免雙重計算);②建築加成是「逐殖民地」、政府加成是「帝國層一次」,兩者範圍不同;③維護費/超支只從加總後扣,不進建築倍率。

## 三、殖民地產出:食物 / 工業 / 研究

殖民地把人口分配為農夫/工人/科學家三類,各自的「每人基礎產出」由行星屬性決定,再疊加建築、種族天賦、士氣、重力。

### 3.1 食物 — 氣候表(每農夫基礎食物,手冊 p.58-59)

| 氣候 | 每農夫食物 | 備註 |
|---|---|---|
| Toxic(劇毒) | 0 | 無法農耕;建築維護 +50% |
| Radiated(輻射) | 0 | 人口上限 −75%;維護 +25% |
| Barren(荒蕪) | 0 | 無土壤 |
| Desert(沙漠) | 1 | 維護 +25% |
| Tundra(凍原) | 1 | |
| Arid(乾旱) | 1 | |
| Ocean(海洋) | 2 | |
| Swamp(沼澤) | 2 | |
| Terran(類地) | 2 | |
| Gaia(蓋亞) | 3 | 樂園級 |

每殖民者每回合吃 1 食物(例外:Lithovore 食岩族不吃、Cybernetic 半機械族吃 0.5)。剩食可外銷換 BC(§2.5)或由運輸艦調度到缺糧殖民地(§7)。remake:`gamedata.ClimateFoodPerFarmer`。

> 考據註:原版存檔的 `food_per_farmer` 欄位以**半單位**儲存(Terran 存 4 = 2.0 食物),與手冊整數表逐字相符;之所以整個食物帳本用半單位,是因為存在 0.5 粒度的消費者(Meklar 每人吃半食物、Natives 農業 +2 等)。工業/研究欄位是整單位,故不受此編碼影響。

### 3.2 工業 — 礦產豐度表(每工人基礎工業,手冊 p.56-57)

| 礦產豐度 | 每工人工業 |
|---|---|
| Ultra-Poor(極貧) | 1 |
| Poor(貧) | 2 |
| Abundant(豐,銀河常態) | 3 |
| Rich(富) | 5 |
| Ultra-Rich(極富) | 8 |

工人產出下限鎖 1/人(手冊 p.20)。remake:`gamedata.MineralIndustryPerWorker`(= `PlanetBaseProduction`,與 openorion2 `gamestate.cpp:31` 交叉驗證)。

### 3.3 研究 — 每科學家基礎產出

**銀河基準 = 3 研究點/科學家**(手冊 p.20 / p.62-63「usual 3」)。此值不依氣候/礦產,是全域固定值。加成:

| 來源 | 效果 | 手冊頁 |
|---|---|---|
| Creative/Psilon 種族 | 每科學家 +2(= 5) | p.16 |
| Artifacts World 種族天賦 | 每科學家 5(取代 3) | p.25 |
| 系統特殊「Artifacts」 | 該星每科學家 +2 | p.60 |

remake:`gamedata.ResearchPerScientistNorm = 3`(先前誤植 30,約 10 倍過高,2026-07-12 已依手冊校正)。

### 3.4 建築加成(加算,不佔人口)

建築提供的產量/研究/食物是**加算(additive)**疊加在人口產出上,不佔人口格位。代表性(手冊 p.78-82):自動工廠(每工人 +1、殖民地 +5)、深核礦場(每工人 +3、殖民地 +15)、研究實驗室、生態圈(人口上限 +2)、土壤改良(每農夫 +1)等。詳見 `docs/tech/colony-buildings.md`。

## 四、重力、人口上限、士氣

### 4.1 重力懲罰(手冊 p.58)

行星重力對農夫/科學家/工人三種產出同時打折:

| 行星重力 | 對一般(Normal-G)種族的產出懲罰 |
|---|---|
| Low-G(低重力) | −25% |
| Normal-G(標準) | 0% |
| Heavy-G(高重力) | −50% |

種族天賦平移基準:Low-G 種族只在低重力星無懲罰、在標準重力受「半個高重力懲罰」;High-G 種族在高/標準重力皆無懲罰。可用重力產生器建築抵銷。remake:`gamedata.gravityPenaltyTable`(NORMAL_G 列 `{−25, 0, −50}` 對齊手冊)。

### 4.2 人口上限(手冊 p.55-59)

由行星**大小**與**氣候**共同決定:

| 大小 | 人口上限範圍(依氣候) |
|---|---|
| Tiny | 1–5 |
| Small | 3–10 |
| Medium | 4–15 |
| Large | 5–20 |
| Huge | 6–25 |

公式(remake `gamedata.PlanetBasePopMax`,移植自 openorion2 `gamestate.cpp:2288`,已與手冊上表逐字驗證):

```
基礎上限 = ((size+1) × 5 × 氣候容量係數% + 50) / 100     (向下取整)
氣候容量係數:Toxic~Ocean 25%、Swamp 40%、Arid 60%、Terran 80%、Gaia 100%
```

上限硬夾 42(1.50 patch note「Population Capacity Clipped To 42」)。額外加成:生態圈 +2、先進都市計畫 +5、Subterranean 種族 +2×(size+1)、Tolerant 種族 +25%、Radiated 星 −75%。

### 4.3 士氣(Morale,手冊 p.65-66 / p.169-170)

每一格士氣圖示(笑臉/哭臉)代表該殖民地**總產出**(食物、工業、研究、收入四者皆適用)± 10%:

```
最終產出 = 基礎產出 × (100 + 淨士氣圖示 × 10) / 100
```

政府基礎士氣、首都淪陷懲罰、娛樂設施等建築都影響士氣(政府部分見 §6)。Unification 政府與機器人(Android)人口**沒有士氣**(恆「知足」),改吃固定的食物/工業加成。remake:`gamedata.MoraleProductionOutput`。

## 五、系統特殊(System Specials)

首次探索某恆星系時,有機率觸發一個「系統特殊」(除太空怪獸外每系統至多一個,手冊 p.60-61)。與經濟/帝國相關者:

| 特殊 | 效果 |
|---|---|
| Gold(金礦) | 該行星殖民地 +5 BC/回合 |
| Gems(寶石) | 該行星殖民地 +10 BC/回合 |
| Artifacts(遺跡) | 該星每科學家 +2 研究;首個發現的帝國另得 1–2 項免費科技 |
| Cache / Debris | 一次性加入國庫的 BC |
| Natives(原住民) | 殖民後併入人口,只當農夫(+2 食物),不帶種族加成、不離開母星 |
| Splinter(失散殖民) | 直接獲得一座小殖民地(3 人口) |
| Hero(英雄) | 免雇用費獲得一名傭兵領袖(仍需付薪資) |
| Wormhole(蟲洞) | 兩星系間航行縮短為 1 回合 |
| Monster(太空怪獸) | 進入即遭攻擊,須以戰鬥清除 |

## 六、政府型態

政府在開局決定、遊戲中不變,但可透過研究升級到同系的進階型態(手冊 p.21-24)。四大系各有取捨:

| 政府 | 主要特徵 | 進階型態 |
|---|---|---|
| **封建(Feudal)** | 造艦成本 2/3、研究只有一半;無兵營時士氣 −20%;同化 8 回合;首都淪陷 → 全帝國無政府 −50% 士氣 | 邦聯(Confederation):造艦 1/3、研究 3/4、同化 4 回合 |
| **獨裁(Dictatorship)** | 防禦間諜 +10;無兵營時士氣 −20%;同化 8 回合;首都淪陷 −35% 士氣 | 帝國(Imperium):指揮評等 +50%、防禦間諜 +15、全殖民地士氣 +20%、同化 4 回合 |
| **民主(Democracy)** | 防禦間諜 −10%;**研究與人頭稅收入各 +50%**(含研究/貿易條約);同化 4 回合;禁止屠殺俘虜人口;首都淪陷 −20% 士氣 | 聯邦(Federation):研究與收入加成升至 +75%、同化 2 回合 |
| **統一(Unification)** | 無士氣機制(建築士氣效果全部無效);**食物與工業 +50%**;防禦間諜 +15%;同化 20 回合;無首都建築、首都淪陷無影響 | 銀河統一(Galactic Unification):食物/工業加成升至 +100%、同化 15 回合 |

經濟相關重點:民主/聯邦是「錢+研究」政府(§2.6 的 money 倍率即出於此);統一是「食物+工業」政府,以固定產出加成取代士氣。remake 目前政府固定為獨裁,政府升級/選擇機制尚未實作;民主/聯邦的 money 加成已接線(`IncomeGovtMoneyBonusPercent`)。

## 七、指揮評等與艦隊維護

**指揮評等(Command Rating)** 是艦隊維護的抽象額度(手冊 p.169)。供需失衡就從國庫扣錢。

### 7.1 供給(帝國能養多少艦)

| 來源 | 供給 | 手冊頁 |
|---|---|---|
| 帝國基礎值 | +5 | — (見下考據) |
| 星基(Star Base) | +1 | p.79 |
| 戰鬥站(Battlestation) | +2(取代星基) | p.82 |
| 星辰要塞(Star Fortress) | +3(取代前二者) | p.83 |
| 通訊科技 / Operations 領袖 | 定性加成 | p.169(remake 未實作) |
| 帝國政府(Imperium) | ×1.5 | p.24(remake 未實作) |

> 帝國基礎值 +5 手冊未明文,由原版存檔 `SAVE10.GAM` 反推:5 名不同種族玩家各持 1 殖民地,扣掉各自軌道衛星加成後一致得基礎 5,與種族/政府無關。remake:`gamedata.CommandPointsBase = 5`。

### 7.2 需求(每艘艦吃多少點)

需求 = 該艦的船體等級(size class):

| 艦體 | 需求 | 維護超支費 |
|---|---|---|
| Frigate 巡防艦 | 1 | 10 BC |
| Destroyer 驅逐艦 | 2 | 20 BC |
| Cruiser 巡洋艦 | 3 | 30 BC |
| Battleship 戰艦 | 4 | 40 BC |
| Titan 泰坦 | 5 | 50 BC(p.83) |
| Doom Star 末日之星 | 6 | 60 BC(p.84) |
| 支援艦(殖民/前哨/運兵) | 1(視同 Frigate) | 10 BC |
| **運輸艦隊(Freighter Fleet)** | **不計** | 手冊明文排除(p.168) |

```
未覆蓋需求 = max(0, 總需求 − 總供給)
超支懲罰   = 未覆蓋需求 × 10 BC/回合    (手冊 p.169)
```

### 7.3 其他維護

| 項目 | 費用 | 手冊頁 |
|---|---|---|
| 使用中的運輸艦 | 每艘 0.5 BC/回合(閒置不計) | p.169 |
| 星基維護 | 2 BC/回合 | p.79 |
| 次元傳送門維護 | 2 BC/回合 | p.106 |

**運輸艦(Freighter)** 專門運糧/運人:每建一次「運輸艦隊」得 5 艘,每艘可自動把 1 單位食物從有餘糧的殖民地送到缺糧殖民地;運人口則需整支 5 艘專用數回合(手冊 p.168)。運輸艦不計指揮評等、不參戰、不上掃描器。

> 版本差異:1.31 建運輸艦隊有「固定回饋 5 BC」的免費現金 quirk,1.50 出廠改為 0(`CHANGELOG_150.TXT` 1.50.8)。remake 以 `RuleProfile.FreightersCashBonus`(1.3=5 / 1.5=0)模擬。

## 八、外交

外交透過「種族關係台(Race Relations console)」進行(手冊 p.170)。當你的船能抵達對方任一殖民地、或對方能抵達你的殖民地時,外交即開啟。可提議締約、貿易/研究條約、科技交換、宣戰、求和,並查詢對方態度。

種族天賦大幅影響外交(手冊 p.15-16 / p.25-26):

| 天賦 | 效果 |
|---|---|
| Charismatic(魅力) | 好的外交行動效果加倍、壞的減半;提案接受率 +50;傭兵較願加入且較便宜 |
| Repulsive(令人厭惡) | 所有種族立即厭惡,外交選項嚴重受限;傭兵較貴且較不願加入 |
| Telepathic(心靈感應,如 Elerian) | 外交協商 +25;可讀取對方動機 |
| Human(人類) | 對任何種族外交 +50%;易同化;雇傭領袖便宜 |

外交關係也決定銀河議會投票時第三方種族把票投給誰(§9)。remake 的外交/AI-對-AI 關係模型仍簡化,詳見 `docs/tech/victory-conditions.md`。

## 九、勝利條件

**原版只有 3 種勝利**(手冊 p.182-183;注意:2016 重製版的 6 種勝利不適用本作):

### 9.1 征服(Conquest)

消滅或迫降所有其他帝國即勝。**佔領 Orion 星系有強力獎勵但不自動獲勝**。

### 9.2 銀河議會選舉(Galactic Council,手冊 p.183)

| 條件 | 門檻 | 明確度 |
|---|---|---|
| 議會成立:銀河殖民率 | 半數星系已被殖民 | ✅ 明確(1/2) |
| 議會成立:存續種族數 | ≥ 3 | ✅ 明確 |
| 當選門檻 | 全體票數的 2/3 超級多數 | ✅ 明確 |
| 人口 → 票數換算 | 依各帝國人口分配票數 | ❌ 手冊無係數(remake 採 1:1 近似) |

運作:議會依各帝國人口分配票數 → 選出票數最高兩位為候選人 → 其餘種族依外交關係投票 → 某候選人得 2/3 即成為銀河統治者、遊戲結束。**當選者若不是你,你可拒絕接受 → 進入全面開戰、遊戲繼續**(手冊:「議會無法強迫你接受不同意的決定」)。

remake:`gamedata.CouncilEligible` / `CouncilVotes` + `engine.CheckHighCouncil`(2/3 用整數 `votesFor×3 ≥ total×2`)。議會重開間隔手冊無數字,remake 排程選 8 回合。

### 9.3 安塔蘭勝利(Antaran Victory,手冊 p.183 / p.106)

研究並建造**次元傳送門(Dimensional Portal)**,派艦隊經傳送門進入次元空間、擊敗安塔蘭母艦隊,其餘種族即臣服。**若開局關閉「安塔蘭入侵」則此路徑不可用**。remake:`internal/shell/antaran_victory.go`(母艦隊戰力為保守預設,手冊/openorion2 均無精確數字)。

> 計分系統(Score Calculation):`MANUAL_150.html` 給出完整計分公式(時間/人口/科技/殲滅/Guardian/Antares/議會各項獎勵,如議會勝 +100 分),但 remake 尚無計分/歷史圖表,僅預先記錄權威值。

## 十、間諜

間諜執行破壞(sabotage)與諜報任務。種族/政府對間諜的加減(手冊 p.20 / p.21-24 / p.25-26):

| 來源 | 效果 |
|---|---|
| Darlok(達洛克)種族 | 任務成功率 +20;可隱形不被遠程偵測 |
| Telepathic 種族 | 間諜任務 +10(不分任務類型) |
| 獨裁政府 | 防禦間諜每次任務 +10 |
| 帝國(Imperium) | 防禦間諜 +15 |
| 統一 / 銀河統一 | 防禦間諜 +15%(無叛徒) |
| 民主政府 | 防禦間諜 −10% |
| 聯邦 | 承襲民主 −10% |

間諜攻防公式(jam chance、Spy vs Spy 判定)以 `MANUAL_150.html`「Notes on Spying」為權威,詳見 `docs/tech/moo2-formulas-reference.md` §14(`spy.go`)。手冊未給精確數字的映射(如軍官技能等級 → 暗殺機率)保留範圍常數、不外插。
